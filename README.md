# Бот-напоминалка о поездках

## Что нового по сравнению с прошлой версией

1. **Токен вынесен из кода** — теперь читается из `.env` через `config.py`. Старый
   токен, который был захардкожен в `main.py`, нужно считать скомпрометированным —
   отзови его в @BotFather (`/revoke`) и впиши новый в `.env`.
2. **Реальный расчёт маршрута через 2ГИС** (`maps.py`):
   - `Geocoder API` превращает введённый пользователем адрес в координаты —
     при создании и редактировании поездки бот сразу проверяет, что место
     существует, и просит уточнить адрес, если не нашёл.
   - `Routing API` (`/routing/7.0.0/global`) считает время в пути на авто/пешком
     с учётом пробок в реальном времени.
   - `Routing API` (`/public_transport/2.0`) считает время в пути на
     общественном транспорте.
3. **Место назначения — точкой на карте или поиском.** При создании и
   редактировании поездки можно либо ввести адрес текстом, либо прислать
   геометку на карте кнопкой «📍 Отправить точку на карте» — бот сам определит
   адрес через обратное геокодирование 2ГИС.
4. **Карта и маршрут одной кнопкой.** После создания поездки, по кнопке
   «🗺 Маршрут» в списке поездок и вместе с самим напоминанием бот присылает
   картинку карты (`Static API` 2ГИС) с точкой назначения и, если известна
   геолокация пользователя, — линией маршрута, а также инлайн-кнопку
   «🧭 Открыть маршрут в 2ГИС» (открывает построенный маршрут в приложении
   2ГИС или на 2gis.ru в браузере, если приложения нет).
5. **Live-геолокация Telegram**. Пользователь делится геопозицией (обычной или
   транслируемой — Live), бот сохраняет последние координаты и обновляет их на
   лету по `edited_message`, пока трансляция активна.
6. **Кэш маршрутов на 3 минуты** в фоновом планировщике — чтобы не дёргать 2ГИС
   каждые 30 секунд по каждой активной поездке без необходимости.
7. Если координат пользователя или пункта назначения нет (например, 2ГИС временно
   недоступен), бот использует грубую оценку как раньше и явно помечает
   напоминание как приблизительное — прозрачность вместо тихого фолбэка.

## Схема БД (миграция автоматическая)

- `users`: `+ last_lat, last_lon, last_location_at`
- `trips`: `+ dest_lat, dest_lon`

Миграция происходит сама при старте (`init_db()`), ничего вручную делать не нужно
даже с уже существующей `trips.db`.

## Установка и запуск

```bash
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# впиши в .env реальный TELEGRAM_TOKEN и DGIS_API_KEY

python3 main.py
```

Ключ 2ГИС: получить демо-ключ можно в личном кабинете https://dev.2gis.com/
(на ключе должны быть включены Geocoder API, Routing API и Static API — без
последнего картинки карт с маршрутом отправляться не будут).

## Известные ограничения / что можно улучшить дальше

- Общественный транспорт: 2ГИС Public Transport API не по всем городам даёт
  одинаково подробное покрытие — в редких случаях маршрут может не построиться,
  тогда используется грубая оценка (см. пункт 5 выше).
- Live-геолокация в Telegram имеет ограничение по времени трансляции (до 8 часов),
  после истечения бот продолжит использовать последние полученные координаты.
- Часовой пояс: время прибытия сравнивается с `datetime.now()` сервера — если бот
  и пользователь в разных часовых поясах, это стоит учесть отдельно.
- Если у ключа не включён Static API (или он временно недоступен), картинка карты
  не отправится, но кнопка «🧭 Открыть маршрут в 2ГИС» всё равно придёт отдельным
  сообщением — сценарий не ломается.
